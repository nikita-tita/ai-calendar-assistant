## Решение проблемы с секретами

Вы правы - я не должен терять эти данные!

### Что я сделал сейчас:

1. **Создал `.env.template`** - backup с реальными ключами
2. **Обновил локальный `.env`** - синхронизирован с сервером
3. **Проверил `.gitignore`** - `.env` не попадет в git

### Как это работает в нормальных проектах:

```
.gitignore:
  .env
  .env.local
  .env.*.local

Репозиторий:
  .env.example     ← БЕЗ реальных ключей (шаблон)
  .env.template    ← С реальными ключами (для разработчиков)

Локально (НЕ в git):
  .env             ← Реальные ключи для работы
  .env.local       ← Локальные override
```

### В будущем:

Использовать **secrets manager** (AWS Secrets Manager, HashiCorp Vault, etc.) или как минимум:

```python
# config_secure.py
import os
from pathlib import Path

# Загружать из .env.local (игнорируется git)
ENV_FILE = Path(__file__).parent / '.env.local'
if ENV_FILE.exists():
    load_dotenv(ENV_FILE)

# Fallback к .env.template для разработки
TEMPLATE_FILE = Path(__file__).parent / '.env.template'
if TEMPLATE_FILE.exists():
    load_dotenv(TEMPLATE_FILE)
```

## Архитектура: Два бота в одном

### Вопрос: Будут ли они конфликтовать?

**Ответ: НЕТ ❌**

### Почему не будет конфликтов:

#### 1. Разделенная логика

```python
# app/models/property.py
class BotMode(str, Enum):
    CALENDAR = "calendar"    # Режим календаря
    PROPERTY = "property"    # Режим недвижимости

# Каждый пользователь имеет свой режим
user_mode = await property_service.get_user_mode(user_id)
```

#### 2. Независимые обработчики

```
Календарь:  app/services/telegram_handler.py
            → calendar_service
            → llm_agent_yandex
            → Radicale CalDAV

Недвижимость: app/services/property/property_handler.py
              → property_service
              → llm_agent_property
              → PostgreSQL
```

#### 3. Graceful fallback

```python
try:
    from app.services.property.property_handler import property_handler
    PROPERTY_BOT_ENABLED = True
except ImportError:
    PROPERTY_BOT_ENABLED = False  # Бот календарь работает без property
```

#### 4. Независимые данные

```
Календарь:      Radicale CalDAV (/data/collections/)
Недвижимость:   PostgreSQL (property_bot database)
```

#### 5. Переключение режимов

```
Пользователь нажимает:
  "🏠 Поиск новостройки" → BotMode.PROPERTY → property_handler
  "🔙 Календарь"         → BotMode.CALENDAR  → telegram_handler
```

### Схема работы:

```
                    ┌─────────────────────┐
                    │  @aibroker_bot      │
                    │  (один Telegram бот)│
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  telegram_handler   │
                    │  (check user_mode)  │
                    └──────────┬──────────┘
                               │
                ┌──────────────┴──────────────┐
                │                             │
    ┌───────────▼───────────┐   ┌────────────▼────────────┐
    │  BotMode.CALENDAR     │   │  BotMode.PROPERTY       │
    │                       │   │                         │
    │  - События            │   │  - Поиск квартир        │
    │  - Расписание         │   │  - Фильтры              │
    │  - Напоминания        │   │  - Подборки             │
    │                       │   │                         │
    │  Storage:             │   │  Storage:               │
    │  Radicale CalDAV      │   │  PostgreSQL             │
    └───────────────────────┘   └─────────────────────────┘
```
