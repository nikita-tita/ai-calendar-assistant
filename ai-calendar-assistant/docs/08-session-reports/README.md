## Правила работы с отчётами

1. **Создание отчёта:**
   - Создать папку с датой: `YYYY-MM-DD-краткое-описание`
   - Добавить markdown файл с детальным отчётом
   - Обновить этот README

2. **Содержание отчёта:**
   - Описание проблемы
   - Причины
   - Решение
   - Изменённые файлы (со ссылками)
   - Результаты

3. **После завершения:**
   - Убедиться, что все временные файлы перенесены в `docs/07-archive/`
   - Корневая директория проекта должна содержать только `README.md`

---

**ВАЖНО:** Все отчёты, заметки и временные документы должны храниться ТОЛЬКО в папке `docs/`. Корневая директория проекта предназначена исключительно для кода, конфигурации и основного README.
